# 긁기 강도 오분류 진단 연구 도구

스마트워치 IMU(가속도+자이로, 100Hz)로 측정한 긁기 강도(약/보통/강)를 **머신러닝 없이 임계값**으로
분류하고, **왜 틀렸는지를 눈으로 진단**하기 위한 연구 분석 도구입니다.

## 방식
1. 특징(기본 진폭 std)의 **약 기준**(약 긁기 평균)·**강 기준**(강 긁기 평균) 두 앵커를 잡음
2. 새 긁기가 두 기준 사이 어디인지 **위치(0~1)** 계산
3. 낮으면 약, 높으면 강, 사이는 보통 (두 경계 슬라이더로 조정)

→ 곱셈/뺄셈 몇 번. "왜 이 판정?"을 특징값으로 바로 설명 가능.

## 실행
```bash
pip install -r requirements.txt
streamlit run app.py
```
샘플 데이터(`sample_data.ndjson`, soonho 32개)로 바로 실행됩니다.

## 검증 (중요)
숫자를 정직하게 보기 위해 **학습·평가 샘플이 겹치지 않도록** 교차검증합니다.
- **LOOCV**: 샘플 1개를 빼고 나머지로 앵커를 학습해 그 1개를 판정 (기본)
- **LOSO**: 피험자 1명을 빼고 학습 (피험자 2명 이상일 때 사이드바에 자동 노출)

모든 지표(정확도·QWK·오분류)는 이 교차검증 결과입니다. 계산은 결정적이며 `np.random.seed(0)` 고정.

## 화면 구성 — 두 창
사이드바: **데이터/평가 업로드 · 분류 특징 · 검증 방식 · 경계 · 필터 · 마지막 1초 제거**. 상단: 요약 지표(정확도·QWK·심각오류·오분류)와 **샘플링 레이트**(t_ns에서 자동 추정, 예: 100 Hz). 아래는 두 창으로 나뉩니다.

### 📊 데이터 종합 (전체를 한눈에)
- **오분류 한눈에 보기(스트립 플롯)**: 가로=위치 점수, 세로=실제 강도 레인, 배경 초록/노랑/빨강=예측 구간. ○=정분류, **✕=오분류**(색=예측). 정분류는 약→강 대각선을 이루고, **대각선을 벗어난 ✕가 오류**(경계 근처=아슬아슬, 약 레인 오른쪽 끝=심각). 호버로 상세.
- 내부 탭 4개: **패턴 요약** / **특징 비교** / **경계 최적화** / **실시간 분류** (세부는 아래)

#### 실시간 분류 (윈도우 단위 시계열 재생)
동작 단위로 잘린 평가 샘플들을 **시간 순으로 이어붙여 가상 연속 신호**를 만들고(경계에 0.5초 무동작 구간 삽입), **1초 윈도우·0.25초 홉**으로 전진하며 각 윈도우를 판정합니다. (헤더에 "실제 연속 녹화 아님" 명시)
- **재생 제어**: ▶ 재생 / ⏸ 일시정지 / ⏭ 한 윈도우 / ⏹ 정지 + 속도 슬라이더 (`st.session_state`로 위치 관리 → 일시정지 후 재개).
- **실시간 갱신**: 흘러가는 파형·스펙트로그램(현재 위치 표시), 현재 윈도우 판정 vs 정답, **누적 정확도(윈도우 기준, n/d 표기)**, 처리 윈도우 수/전체, **윈도우당 지연(ms)** — 1초/0.25초 홉이면 250ms 예산 안이 실시간 가능 기준.
- **타임라인 비교 띠**: 위=정답·아래=예측을 시간축에 정렬해, **틀린 구간이 어디인지** 한눈에. 재생하며 왼쪽부터 채워짐.
- **분류 대상**(사이드바): `강도 측정`(규칙기반 앵커) / `긁기 감지`(RandomForest, SCRATCH/NOT_SCRATCH) / `파이프라인`(SCRATCH 윈도우만 강도 판정).
- **모델 연동**: 평가용 데이터를 올리면 **학습 데이터로만** 학습한 모델(강도=앵커/표준화, 감지=RF)을 적용(정직한 out-of-sample). 없으면 현재 데이터로 재생하며 참고용 표기.
- **긁기 감지 모델**: 가속도·자이로 크기 각각의 std·MAD·범위·저크·지배주파수·자기상관피크(12개) → `RandomForestClassifier(n_estimators=150, class_weight='balanced', random_state=42)`. **동작 단위 그룹 분리(GroupKFold)** 로 같은 동작의 윈도우가 학습·평가에 겹치지 않게 CV 정확도 계산.
- **데이터 누수 경고**: 학습·평가가 같은 파일이거나 `subject_id`가 겹치면 ⚠️ 과대평가 경고, 다른 피험자면 ✅ LOSO 안내, 평가 미업로드면 참고용 캡션.
- **라벨 없는 데이터**: 판정만 표시하고 정확도 영역엔 "정답 라벨 없음".

### 🔬 개별 상세 (한 긁기를 시간축으로)
샘플 하나를 고르면 **실시간 분류처럼 그 한 긁기를 시간축으로 윈도우 분석**합니다(1초 창/0.25초 홉). 강도 모델은 **이 샘플(LOSO면 이 피험자)을 제외하고 학습**해 정직한 out-of-sample 흐름을 봅니다.
- **메인: 강도 흐름 차트** — x=시간, y=위치점수(0=약 기준·1=강 기준), 배경을 약(초록)/보통(주황)/강(빨강) **구간으로 칠해** 선이 어느 강도로 흐르는지 한눈에. 각 윈도우 마커는 예측 강도 색, **○=정답 라벨과 일치·✕=불일치**. 아래 예측 리본으로 색맹 대비 이중 표기. → *"어느 강도로 흐르나"* 와 *"얼마나 정확/일관적인가"* 를 한 그림에서.
- **KPI**: 최종 판정(다수결) · **시간 일관성%**(최빈 예측 비율=판정이 안 흔들리나) · **정답 일치율%**(예측==실제 창 비율) · 윈도우 수. 일관성과 일치율을 분리해 "안정적으로 틀림" vs "경계 깜빡임"을 구분.
- **재생 ▶/⏸/⏭/⏹ + 속도**: 커서가 시간축을 지나며 왼쪽부터 채워지고, 일시정지하면 그 윈도우의 파형·스펙트로그램을 드릴다운.
- **접이식(expander)로 정리**: 원신호(파형·스펙트로그램·PSD·자이로) / 특징값 17종·특징 시간변화 / 누적 일치율 곡선·판정 로그 / (오분류면) 같은 라벨 정분류와 나란히 비교(스펙트로그램·PSD·특징 차이).

### 전처리: 마지막 1초 제거
- 사이드바 토글(기본 ON). 긁기 종료 구간의 잡음/여운을 제거하기 위해 각 신호 끝에서 **(1초 × 샘플링레이트)** 샘플을 잘라낸 뒤 특징을 계산합니다. (100 Hz면 끝 100샘플)
- 끄면 원본 전체로 계산 — 켜고 끄며 성능 차이를 비교할 수 있습니다.

### 2. 패턴 요약
- **혼동행렬**(약/보통/강)
- **조건별 오분류율 막대**(부위·사람·방식·자세) → 어디서 많이 틀리나
- **부위 × 강도 정확도 히트맵** → 집중 오분류 구간

### 3. 특징 비교 (연구용)
- 분류 특징 = **진폭(std)·저크·스펙트럼 중심(Hz)·자이로 std** (기본 선택 `jerk`, 단일 특징 QWK 최고).
  - `energy` 제거(=std²로 진폭과 상관 1.00, 중복), `peak_freq`(불안정) → **`spectral_centroid`**(Welch PSD 무게중심, 안정적)로 교체.
- **Spearman rho**(강도 순서와의 상관) 순위 + 단일특징 QWK/정확도 표
- 특징별 **박스플롯 그리드**(약/보통/강 분포 비교, 선택 특징 ⭐)
- 참고 문구: 진폭·저크·MAD·에너지는 서로 상관 0.95↑ — 하나만. **진폭(또는 저크) + 스펙트럼 중심**이 QWK 0.89로 최고(크기와 속도, 서로 다른 축).

#### 여러 특징 함께 쓰기 (다차원 앵커 투영, 무학습)
사이드바에서 특징을 **여러 개 선택**하면, 단일 특징과 같은 논리를 다차원으로 확장합니다:
- 각 특징을 **표준화(z-score)** → 약 긁기 중심 벡터(약 앵커)·강 긁기 중심 벡터(강 앵커)
- 새 긁기를 **약→강 중심축에 투영**해 위치(0~1) → 기존 경계 규칙 그대로 적용
- 축 방향을 *학습*하지 않으므로 **여전히 머신러닝 아님**(투명). 위치가 1D라 오분류 진단·경계·검증 전부 그대로 동작
- **경계는 1차원이 아니게 됩니다**: 특징 2개면 평면상의 **직선 2개(띠)**, 3개면 평면 2개(초평면). 특징을 정확히 2개 고르면 이 2D 결정 경계를 그림으로 보여줍니다.
- "조합 QWK vs 최고 단일 QWK"를 함께 표시해 **여러 특징이 실제로 도움 되는지 데이터로 판단**(도움 안 되면 단일 특징 권장)
- 참고: 축 방향까지 *학습*하려면 LDA/로지스틱회귀인데, 그건 이 프로젝트가 배제한 머신러닝입니다.

### 4. 경계 최적화
- **range 슬라이더**(양쪽 손잡이)로 약↔보통·보통↔강 경계를 지정 — 0.5에 얽매이지 않고 0~1 어디든(약 경계 ≤ 강 경계만 유지). 0.01 단위 미세 조정.
- **여러 기준(정확도·macro-F1·QWK)으로 최적 경계 자동 탐색**(전 구간, 두 경계가 둘 다 0.5 미만/초과여도 탐색) + **심각 오류**(약↔강 정반대) 개수
- 강도는 순서형이라 **QWK 권장**, 경계별 QWK 히트맵 제공

## 필터
`subject_id / body_region / method / posture` 중 **값이 여러 개인 축이 자동으로 필터에 노출**되고,
모든 분석이 필터된 데이터 기준으로 갱신됩니다. 지금은 데이터가 한 사람뿐이라 부위만 뜨지만,
여러 사람 데이터가 오면 사람·방식·자세 필터와 LOSO 검증이 자동으로 활성화됩니다.

## 데이터 형식 (NDJSON, 한 줄 = 한 긁기)
```json
{"label":"SCRATCH","intensity":"weak|normal|strong",
 "method":"arm|finger","body_region":"head_neck|arm|torso|leg","posture":"supine|side",
 "subject_id":"P01","accel":{"x":[...],"y":[...],"z":[...],"t_ns":[...]},
 "gyro":{"x":[...],"y":[...],"z":[...],"t_ns":[...]}}
```
`gyro`가 없으면 자이로 특징은 자동으로 제외됩니다. 사이드바에서 새 NDJSON을 올리면 즉시 그 데이터로 분석합니다.

## 참고 (soonho 32개, 기본 경계 [0.4, 0.6], LOOCV)
- 자동 추천 특징 = **저크**(|rho| 최대). 특징을 바꾸면 성능이 달라지므로 "특징 비교" 탭에서 근거 확인.
- 지표는 교차검증이라 같은 샘플로 학습·평가하던 방식보다 값이 보수적으로(정직하게) 나옵니다.
